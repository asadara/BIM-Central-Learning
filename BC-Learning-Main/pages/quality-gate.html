Quality gate untuk memastikan data BIM layak digunakan

Used for Site/Decision

Quality gate memastikan data hanya dipakai setelah approved

Quality BIM itu bukan bikin model bagus, tapi memastikan model layak dipakai.

Ringkas

  • Gate = titik "boleh dipakai / belum"
  • Kriteria sederhana sebelum dipakai
  • Siapa cek apa (tanpa jadi HR/jobdesc)

Apa yang dicek di gate

Konsistensi, koordinasi, versi, status

  • Konsistensi model dengan spesifikasi
  • Koordinasi antar disiplin
  • Versi terbaru dan authorized
  • Status approval jelas

Output gate

Release note/approval/daftar perubahan

  • Release note dengan versi dan tanggal
  • Daftar perubahan yang disetujui
  • Approval dari responsible party
  • Distribusi ke stakeholder yang perlu

Stop-rule

Jika belum lewat gate → jangan dipakai

  • Data draft tidak boleh untuk procurement
  • Model unreviewed tidak untuk konstruksi
  • Informasi tanpa approval tidak valid
  • Pelanggaran gate dapat cause rework dan claim

Contoh Kasus Proyek

Drawing/shop/4D dipakai lapangan sebelum gate → rework

Dalam proyek gedung, shop drawing yang masih draft digunakan untuk fabrikasi material. Akibatnya, setelah review ditemukan kesalahan dimensi yang menyebabkan rework 20% dari total pekerjaan, dengan biaya tambahan dan delay schedule 2 minggu.

Checklist Cepat

  • Verifikasi status data sebelum penggunaan
  • Pastikan versi terbaru dan konsisten
  • Cek approval dari yang berwenang
  • Simpan release note untuk audit
  • Distribusi informasi sesuai stakeholder
  • Tanggal berlaku jelas dan dikomunikasikan
  • Monitor penggunaan data post-gate
