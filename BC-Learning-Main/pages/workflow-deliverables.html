Deliverables bukan akhir proses—deliverables adalah awal penggunaan informasi untuk konstruksi dan operasi.

Ringkas

  • Model BIM adalah fondasi semua deliverables
  • Drawings dan sheets dihasilkan dari model
  • Schedules terintegrasi dengan model untuk 4D
  • Reports berisi analisis dan dokumentasi
  • As-built record mendokumentasikan kondisi akhir

Hierarki Deliverables

Pijakan: ISO 19650 (information deliverables & container management; cite generally); Indonesia: standar dokumen teknis proyek (SNI/kontrak/PUPR)

Deliverables BIM tersusun dalam hierarki yang terstruktur dari model hingga dokumentasi akhir.

  • Model: Representasi digital 3D sebagai sumber utama informasi
  • Drawings/Sheets: Dokumentasi visual 2D dari model
  • Schedules: Timeline dan sequence konstruksi
  • Reports: Analisis, kalkulasi, dan dokumentasi teknis
  • As-built Record: Dokumentasi kondisi akhir konstruksi

Release Package Management

Pijakan: ISO 19650-2 (information delivery control); Indonesia: SNI 8435:2017 (BIM Data Management)

Package release memastikan deliverables lengkap dan siap digunakan.

  • Version Control: Tracking perubahan dengan nomor versi
  • Metadata: Informasi pendukung (author, date, approval)
  • Quality Check: Verifikasi sebelum release
  • Distribution: Pengiriman ke stakeholder yang tepat

As-built Documentation

Pijakan: ISO 19650-2 (asset information delivery); Indonesia: praktik dokumentasi as-built (verifikasi internal)

As-built record mendokumentasikan perbedaan antara desain dan konstruksi aktual.

  • Model Updates: Update model berdasarkan kondisi lapangan
  • Change Documentation: Catat semua perubahan dari baseline
  • Photo Records: Dokumentasi visual progres konstruksi
  • Commissioning Data: Data pengujian dan commissioning

Contoh Kasus Proyek

Dalam proyek gedung apartemen, model MEP awal diupdate selama konstruksi ketika ditemukan utilitas bawah tanah yang tidak terdeteksi. Drawings direvisi, schedule diadjust, reports analisis clash didokumentasikan. As-built record mencatat semua perubahan untuk referensi maintenance dan warranty.

Checklist Cepat

  • Pastikan model akurat sebelum generate deliverables
  • Sertakan metadata lengkap pada setiap file
  • Verifikasi approval sebelum release package
  • Distribusikan deliverables ke stakeholder tepat waktu
  • Update as-built record secara berkala
  • Simpan semua versi deliverables untuk audit trail
  • Dokumentasikan perubahan dari baseline desain
