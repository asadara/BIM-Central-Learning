Koordinasi bukan kompetisi—koordinasi adalah kolaborasi untuk mencapai hasil terbaik bersama.

Ringkas

  • Issue ditemukan melalui clash detection atau review manual
  • Owner ditugaskan berdasarkan disiplin dan tanggung jawab
  • Solusi diusulkan dan didiskusikan antar stakeholder
  • Agreement dicapai sebelum implementasi perubahan
  • Model diupdate dan direcheck untuk memastikan resolusi
  • Issue ditutup dengan dokumentasi lengkap

Proses Manajemen Issue

Pijakan: ISO 19650-2 (collaboration/coordination workflow; info exchange/approval concept); Indonesia: prosedur RFI/koordinasi proyek (PUPR/kontrak)

Manajemen issue terstruktur memastikan koordinasi efektif antar disiplin.

  • Detection: Issue ditemukan melalui automated tools atau manual review
  • Assignment: Owner ditentukan berdasarkan kompetensi dan workload
  • Resolution: Solusi dikembangkan dengan konsultasi lintas disiplin
  • Implementation: Perubahan diterapkan dengan kontrol versi

Tools Kolaborasi

Pijakan: ISO 19650-2 (information exchange mechanisms); Indonesia: SNI 8440:2017 (Information Exchange)

Tools modern memfasilitasi koordinasi real-time antar tim.

  • CDE Platforms: Centralized document management
  • Clash Detection: Automated interference checking
  • Meeting Tools: Virtual coordination meetings
  • Issue Trackers: Digital issue management systems

Metode Integrasi Model

Pijakan: ISO 19650 (information container integration); Indonesia: SNI 8434:2017 (BIM Model Production)

Integrasi model memastikan konsistensi dan akurasi informasi.

  • Federated Model: Model terpisah dengan koordinasi referensi
  • Integrated Model: Single model dengan worksets/sharing
  • Linked Models: Model induk dengan sub-model terkait
  • IFC Exchange: Standard format untuk interoperability

Contoh Kasus Proyek

Dalam proyek mall, clash detection menemukan pipa HVAC menembus kolom struktur. Issue diassign ke MEP lead, yang mengusulkan rerouting pipa. Setelah diskusi dengan structural engineer dan mendapat approval dari BIM coordinator, model diupdate. Recheck memastikan tidak ada clash baru, dan issue ditutup dengan dokumentasi foto instalasi.

Checklist Cepat

  • Jalankan clash detection mingguan
  • Assign issue owner dalam 24 jam
  • Dokumentasikan semua diskusi koordinasi
  • Peroleh approval sebelum implementasi perubahan
  • Update model dengan kontrol versi
  • Lakukan recheck setelah resolusi
  • Simpan semua dokumentasi issue
