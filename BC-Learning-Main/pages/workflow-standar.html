Standar bukan hambatan—standar adalah fondasi untuk konsistensi dan kepercayaan.

Ringkas

  • Status informasi menentukan kapan boleh digunakan
  • Naming dan versioning memastikan traceability
  • Approval gates mencegah penggunaan informasi yang belum siap
  • Release package minimum memastikan kelengkapan deliverable
  • Arsip memelihara history untuk audit dan pembelajaran

Status & Penggunaan Informasi

Pijakan: ISO 19650-2 (information delivery & CDE workflow concepts); Indonesia: SNI ISO 19650-2 (Asset Delivery Phase)

Status informasi menunjukkan kesiapan dan kepercayaan data untuk digunakan dalam pengambilan keputusan.

  • Draft/WIP: Dalam pengerjaan, belum siap untuk sharing
  • Review/Shared: Dibagikan untuk review, feedback diterima
  • Approved/Published: Disetujui, siap digunakan untuk keputusan
  • Archived: Disimpan untuk referensi, tidak lagi aktif

Naming & Version Rules

Pijakan: ISO 19650-2 (information container management); Indonesia: SNI 8435:2017 (BIM Data Management)

Konvensi naming dan versioning memastikan informasi mudah ditemukan dan dilacak perubahannya.

  • Naming Convention: ProjectCode_Discipline_Element_Version.ext
  • Version Control: Gunakan angka sequential (01, 02, 03...)
  • Metadata: Sertakan author, date, status dalam properties

Release Package Minimum

Pijakan: ISO 19650-2 (information deliverables); Indonesia: SNI 8434:2017 (BIM Model Production)

Package release harus memuat semua elemen yang diperlukan untuk penggunaan yang dimaksud.

  • Model Files: IFC atau native format dengan LOD sesuai
  • Documentation: Drawing, schedule, specifications
  • Metadata: Status, approval, change log
  • Supporting Data: QA reports, clash detection results

Contoh Kasus Proyek

Dalam proyek jembatan, model struktur awal (Draft_v01) direview oleh konsultan. Setelah revisi berdasarkan feedback (Shared_v02), model disetujui owner (Approved_v03). Model ini digunakan untuk procurement dan konstruksi. Versi final (Archived_v05) disimpan dengan as-built data untuk maintenance.

Checklist Cepat

  • Tetapkan konvensi naming di awal proyek
  • Gunakan sistem versioning yang konsisten
  • Peroleh approval sebelum publish informasi
  • Verifikasi kelengkapan package sebelum release
  • Arsipkan informasi dengan metadata lengkap
  • Latih tim untuk memahami status informasi
  • Audit penggunaan informasi secara berkala
